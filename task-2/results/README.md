# Task 2

| Критерий                                 | Spring Batch               | Apache Airflow                  | Kubernetes CronJob                | Apache Spark                       |
| ---------------------------------------- |----------------------------|---------------------------------|-----------------------------------|------------------------------------|
| CRON-расписание                          | Поддерживает               | Поддерживает                    | Поддерживает                      | Не поддерживает                    |
| Сложность логики обработки данных        | Низкая (прямолинейные ETL) | Средняя (поддержка сложных DAG) | Очень низкая (скрипт или команда) | Высокая (распределённая обработка) |
| Ресурсоемкость                           | Средняя                    | Средняя                         | Низкая (легковесный под)          | Высокая                            |
| Масштабируемость                         | Хорошая                    | Хорошая                         | Средняя                           | Отличная                           |
| Развёртывание в облаке                   | Средняя сложность          | Средняя сложность               | Просто (Kubernetes нативно)       | Сложно                             |
| Интеграция с логированием и мониторингом | Хорошая                    | Отличная                        | Ограничена возможностями k8s      | Хорошая                            |

- Исходя из низкого объёма обрабатываемых данных, отсутствия дополнительной логики обработки выгружаемых данных, а также
  уже имеющегося кластера k8s логично использовать k8s CronJob.
- Такое решение позволит наиболее быстрым способом решить поставленную задачу без привлечения дополнительных ресурсов и
  разворачивания дополнительного программного обеспечения.
- Так как задача не является распределённой и не требует сложного взаимодействия с различными сервисами, а только
  выгрузку данных из БД - дополнительный мониторинг и трейсинг не видится необходимым.
- Исходя из заявленного объёма обрабатываемых данных текущее решение может без проблем пережить кратный рост
  обрабатываемых данных, прежде чем потребуется применять распределённую разработку в каком-либо виде.
- В случае добавления более комплексной логике обработки данных, следует рассмотреть внедрение Apache Airflow.
